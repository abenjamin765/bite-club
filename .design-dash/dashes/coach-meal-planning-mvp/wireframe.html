Coach-Led Meal Planning MVP - Mobile Wireframe

Mobile-only MVP wireframe set. Primary client screen switches by weekly plan status: approved → Food Log hub; not approved → Planning hub.

Screen Flow Map

Client (S1–S3, S6):
  1. Auth → Plan hub (draft) → Day/meal editor → Submit plan → Awaiting approval (locked)
  2. Plan approved → Food Log hub → Mark meals (Unselected → Followed/Modified/Skipped) → Submit check-in → Confirmation
  3. Reuse: Plan hub → Add from saved → Meal copied into day
Coach (S4–S5):
  1. Auth → Group dashboard → Review submitted plan → Approve
  2. Digest notification → Daily digest → Client check-in detail (planned vs actual)

Client App Shell (mobile-only)

layer: page | object: account | component-hint: top app nav | ui-rule: NAV_PRIMARY,ORG_SHOW_ESSENTIAL
Navigation contract: left chip = profile/group context, center = mode label (Food Log / Plan) with the active week date range beneath it (e.g. Jun 28 – Jul 06), right chip = toggle Plan vs Food Log. The mode label names the current screen; the week range is context, not identity. Day-of-week selector sits below and scrolls horizontally across all 7 days (inactive days meet 4.5:1 contrast). When the week is not approved, Food Log is locked and Plan mode is primary (right chip still available where applicable).

Goal-Page Map (SC-1 to SC-6)

  • SC-1: Food Log meal cards + sticky submit — "Mark all followed" completes the happy path in 1 tap + submit; deviations marked per meal.
  • SC-2: Food Log hub on approved week; check-in submitted confirmation.
  • SC-3: Coach Group dashboard digest summary — one daily view, exception-sorted.
  • SC-4: Full loop screens present: Plan → Submit → Approve → Food Log → Digest.
  • SC-5: No peer data on client surfaces; coach-only group views.
  • SC-6: Modified/Skipped inline on client; planned vs actual on coach check-in detail.

P7 Checklist Snapshot

  • Mobile-first layouts only; current orange/cream styling preserved.
  • Primary screen contract: approved week → Food Log; unapproved → Plan.
  • Header shows mode label (Food Log / Plan) + week-range context; auth shows product name, no phantom week.
  • No pre-answered adherence: meals start Unselected; submit disabled until all marked; "Mark all followed" fast-path for the happy path.
  • Meal card matrix: Unselected, Followed, Modified, Skipped — Skipped and Modified both capture deltas the coach detail displays.
  • Macro tiles labeled "Planned"; actual/delta shown only on Modified.
  • Meal card links out to planned ingredients (no Isolated Object).
  • Weekday strip scrolls all 7 days; inactive-day text meets WCAG 1.4.3 (≥4.5:1).
  • Client flow screens: Auth, Plan draft, Day editor, Submitted locked, Food Log (default / modified / skipped / completed), Confirmation.
  • Coach flow screens: Group digest, Plan review, Client check-in detail.
  • Edge states rendered for Food Log, Plan Wizard, Coach Dashboard, Plan Review, Auth.
  • All interactive sections cite semantic ui-rule IDs; no ui-rule: TBD.
