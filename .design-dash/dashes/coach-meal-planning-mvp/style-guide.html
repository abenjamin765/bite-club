Meal Plan — Style Guide

Production design system for the shipping PWA. Native iOS patterns with a warm food-forward identity: sunrise gradient, rounded display type, plate-check monogram, progress rings, meal chips, and Untitled UI line icons.

Live iOS UI Kit Open app Wireframes tokens.css app.css

Visual direction

The app should feel like a built iOS app — now with a distinct warm brand layer. Content sits on a warm paper canvas, elevated cards stay bright, and signature sunrise accents guide priority actions.

  • Platform feel: iOS Human Interface Guidelines — large titles, blurred nav/action bars, tab bar navigation
  • Brand: Monochromatic orange scale from ColorSpace #FF6B00 + amber support accent (#FFB100) for the sunrise gradient
  • Semantic colors: Green = followed, orange = deviated, red = missed/skipped
  • Density: 44px minimum touch targets; 17px body text; generous grouped spacing
  • Icons: Untitled UI Icons — free line set (1,100+ SVGs, MIT licensed). Inlined in HTML; no runtime CDN dependency.

Layout shell

Every app screen uses body.ios > .ios-viewport (max 440px). Safe-area insets apply automatically.

  • Nav bar — sticky, blurred; 44px inner height + safe-area top
  • Large title — scrolls with content below nav bar
  • Grouped content — .list-group with 16px horizontal inset
  • Tab bar — fixed bottom for primary navigation (client: Today / Plan; coach: Dashboard / Digest)
  • Action bar — fixed bottom for primary CTA (Submit, Approve); use .ios-scroll.has-actionbar

Icons

All UI icons come from the free Untitled UI line set (MIT licensed, commercial use OK). SVGs are inlined in HTML — copy paths from untitledui.com/free-icons or the Live iOS UI Kit icon grid.

Usage rules

  • ViewBox: Always 0 0 24 24
  • Stroke: stroke="currentColor", stroke-width="2", round caps/joins
  • Decorative: Add aria-hidden="true" when adjacent text conveys meaning
  • Sizing: Tab/row icons inherit from parent; nav chevron .chev = 20×20px; row chevron .row-chevron = 16×16px
  • Color: Icons inherit currentColor — use .row-icon.is-green / .is-red / .is-gray containers for semantic tint

Accessibility

  • Minimum 44×44px touch targets on all interactive elements
  • Segmented controls use role="group" with per-meal aria-label
  • Status communicated by text + color (pills), not color alone
  • Decorative icons use aria-hidden="true"; row/tab labels carry the accessible name
  • Form errors use .helper-text.is-error with aria-live="polite"
  • Safe-area insets via env(safe-area-inset-*) on nav, tab, and action bars
  • Neutral deviation language: “Modified” / “Skipped” — no shame framing
